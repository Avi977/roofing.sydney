## Spacing Scale

Derived from Tailwind v4 utility classes observed in `roof-editor.tsx` and `preview/page.tsx`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap (gap-1), inline tight padding |
| sm | 8px | Button vertical padding (py-2), small gaps (gap-2) |
| md | 16px | Horizontal button padding (px-4), section gap |
| lg | 24px | Section spacing (px-6, gap-6) |
| xl | 32px | Layout padding (py-8) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets for the "Load aerial photo" button must be at minimum 44px tall (py-3 + text-sm achieves this — use `py-3 px-5` not `py-1.5 px-3`).

## Typography

Derived from utility classes in `roof-editor.tsx` and `preview/page.tsx`:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / hint | 14px (text-sm) | 400 (font-normal) | 1.5 |
| Label / button | 12px (text-xs) | 600 (font-semibold) | 1.5 |
| Subheading | 16px (text-base implicit, text-sm font-medium) | 500 (font-medium) | 1.5 |
| Heading (page) | 24px / 30px responsive (text-2xl sm:text-3xl) | 600 (font-semibold) | 1.25 |

Phase 1 new elements use text-sm (14px) / font-medium (500) for the button label to match the existing overlay pill pattern (`text-xs font-medium`). The "Load aerial photo" button is a primary action and uses font-semibold (600).

## Color

Derived from `src/app/globals.css` CSS variables:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (--background) | Page background, modal/overlay fills |
| Secondary (30%) | `#f9fafb` (--surface) | Card container (`bg-surface`), image placeholder background |
| Accent (10%) | `#0ea5e9` (--accent) | Focus rings (--ring) only — NOT used for the primary button in this phase |
| Brand / primary | `#0b0f19` (--brand) | Primary button background; matches existing `bg-brand` pattern in reset button and overlay pill |
| Destructive | `#dc2626` (red-600 Tailwind default) | Not present in this phase — no destructive actions |
| Muted text | `#6b7280` (--muted) | Secondary descriptive text, address display, sub-labels |
| Border | `#e5e7eb` (--border) | Card borders, separator lines |

Accent (`#0ea5e9`) reserved for: focus ring only (`ring-accent/35`). It is NOT used for buttons or loading indicators in this phase — all interactive elements use `bg-brand` (#0b0f19) consistent with the existing `bg-brand` button pattern.

## Interaction States

This section is specific to Phase 1's new pre-load flow. The `RoofEditor` component gains a new `imageState` track:

| State | What the user sees |
|-------|-------------------|
| `idle` (pre-load) | Grey placeholder box (`aspect-[4/3] w-full sm:aspect-[16/10] bg-surface rounded-none`) with centred "Load high-res aerial photo" button + sub-label. No map. |
| `loading` (fetching Nearmap) | Placeholder box persists; full-overlay spinner pill appears (`absolute inset-0 flex items-center justify-center bg-background/60 backdrop-blur-sm`) matching the existing "Detecting your roof…" pattern. |
| `loaded` (image ready) | Static `<img>` fills the same bounding box. Transparent `<canvas>` overlaid (`absolute inset-0 h-full w-full`). Instruction hint pill at bottom: "Click the centre of your roof". |
| `error` (Nearmap failed) | Error overlay matching existing pattern. MapLibre map rendered as fallback behind if feasible (D-03 Claude's discretion). |
| `fallback` (MapLibre shown) | Existing MapLibre map view, with subtle label strip: "Showing standard map tiles instead" in muted text at bottom of container. |

Button sizing for "Load aerial photo": `rounded-xl bg-brand px-5 py-3 text-sm font-semibold text-white` — uses the same `bg-brand` + `rounded-xl` + `text-white` pattern as the existing "Try again" button, scaled up to a primary action size (py-3 px-5 instead of py-1.5 px-3).

Image display: `<img>` element fills `aspect-[4/3] w-full sm:aspect-[16/10]` — same dimensions as the MapLibre container it replaces. `object-fit: cover`, no border-radius (container provides `rounded-2xl overflow-hidden` from preview page).

Cursor: `cursor-default` during `idle` and `loading` states. `cursor-crosshair` when image is loaded and status is `preview` (ready for click-to-segment). `cursor-wait` during `processing`.
