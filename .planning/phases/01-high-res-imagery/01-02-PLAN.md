| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | Nearmap image URL in client | accept | Public Supabase bucket URL is intentional — images are non-sensitive aerial views. URL is an opaque Supabase-generated path. |
| T-02-02 | Tampering | Click coordinates passed to segmentAt | accept | Click coords are DOM pixel values bounded by the element's bounding rect — no external injection possible. Worst case: incorrect fill region selected. |
| T-02-03 | Tampering | crossOrigin="anonymous" on <img> | mitigate | Required for createImageBitmap to work without canvas taint when image is from different origin (Supabase CDN). Without this attribute, reading canvas pixel data would throw a security error. Must be present. |
| T-02-04 | Information Disclosure | Supabase URL leaks in network tab | accept | Expected — public URL is the delivery mechanism. No credentials or PII in URL. |
| T-02-05 | Elevation of Privilege | User triggers repeated Nearmap charges via button clicks | mitigate | Button is disabled (imageState is "loading") while fetch is in progress — cannot double-click. Cache-first in API route means same address never charges twice. |
